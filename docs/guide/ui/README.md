# UI Guide Subindex

Status: ACTIVE  
Owner: POS Platform  
Last Reviewed: 2026-02-22

## 1. Purpose

This subfolder groups UI-specific supporting guides so `docs/guide` stays clean and canonical flow docs remain easy to scan.

These docs support implementation and review but do not override canonical flow authorities in `docs/guide/README.md`.

## 2. Included Files

1. `UI_SOT.md`
2. `UI_REVAMP_ROADMAP.md`
3. `UI_AUTOMATION_GUIDE.md`
4. `UI_CONFORMANCE_MATRIX.md`
5. `Clearance CSS Alignment Rules.md`
6. `Remit_Cleanup_Checklist.md`
7. `ROUTE_CLEANUP_CHECKLIST.md`

## 3. Usage Order

1. Start with `UI_REVAMP_ROADMAP.md` for monthly rollout ownership, active wave tracking, and done/summary rules.
2. Use `UI_SOT.md` for canonical UI rules, route-family contracts, and shared-component expectations.
3. Use `UI_AUTOMATION_GUIDE.md` for contract and gates.
4. Use `UI_CONFORMANCE_MATRIX.md` for route-level conformance status.
5. Use `ROUTE_CLEANUP_CHECKLIST.md` to track per-route cleanup completion (`DONE`/`PENDING`).
6. Use `Clearance CSS Alignment Rules.md` and `Remit_Cleanup_Checklist.md` for focused implementation checks.

## 4. Execution Pairing

1. Operational run commands: `docs/automation/runbooks/`
2. Prompt templates: `docs/automation/templates/UI_AUTOMATION_PROMPT_TEMPLATE.md`
